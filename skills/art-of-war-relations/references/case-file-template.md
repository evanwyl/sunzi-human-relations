# Person case file template

```markdown
# [人物标题]

- slug: coworker-xiaowang
- relation: coworker
- display_name: 小王
- aliases:
  - 那个同事
  - 王哥
- status: active

## Relationship background
- 你和这个人的关系定位
- 当前处于什么阶段
- 是否存在持续冲突 / 拉扯 / 模糊地带

## Known patterns
- 对方常见说话方式
- 对方常见策略 / 回避方式 / 试探方式
- 对方在压力下通常怎么反应

## Verified effective responses
- 哪些回应方式有效
- 为什么有效

## Verified bad moves
- 哪些回应容易踩雷
- 为什么会踩雷

## Key interactions
### [日期] 事件标题
- Situation:
- What happened:
- Surface issue:
- Underlying conflict:
- Suggested move:
- Why this move:
- Strategy basis:
- What to watch next:

## Current reading
- 当前最合理的局势判断
- 主动权在哪边
- 现在最该防什么
- 下次优先测试什么
```
```

## Notes

- `Known patterns` 写重复模式，不写一次性情绪
- `Verified effective responses` 只写已经验证过的方式
- `Verified bad moves` 能帮后续避免重复踩坑
- `Current reading` 可以更新，不是永远不变
